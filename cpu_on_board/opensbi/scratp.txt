This will crash OpenSBI. OpenSBI runs in Machine Mode (M-mode). The RISC-V specification states that M-mode always uses physical addresses, even if Linux (S-mode) has turned the MMU on! Only S-mode and U-mode use the MMU.

4. OpenSBI requires PMP Registers

You commented out the PMP (Physical Memory Protection) registers. During boot, OpenSBI will read and write pmpcfg0 and pmpaddr0 to figure out your memory protection. If they are missing, OpenSBI will crash.

The Fix:
You don't have to actually implement memory protection logic, you just have to give OpenSBI the registers to read/write!

Increase your CSR array size: (* ram_style = "logic" *) reg [63:0] Csrs [0:36];

Uncomment the PMP registers in your case(w_csr) block so OpenSBI can successfully write to them and read them back.
